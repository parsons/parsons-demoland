JavaScript The Document Object Model

When you open a web page, your browser retrieves the page's HTML text, and builds up a model of the document's structure and uses it to render the page. This is the Document Object Model, or DOM. JavaScript can do all sorts of things to our web pages by accessing and manipulating this model.

When interacting or manipulating HTML elements, JavaScript comes in as an "afterthought," meaning that the script would take place after the initial HTML data, and then the DOM updates the visual display.

That is why we need to put the pair of "scrip" tags just before the closing "/body" tag, since JavaScript needs the elements to be loaded and ready.

A couple things that we can do in browser inspectors: You can examine the DOM structure under "Elements" tab, and modify DOM under the "Console" tab.
