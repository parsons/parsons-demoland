JavaScript DRY in JavaScript

The same DRY principle applies to JavaScript. In this example, we have three cards that all do the same thing when clicked — but the code is written out three separate times.

If you wanted to change the click color from coral to steelblue, you'd have to update it in three places. And if you added a fourth card, you'd have to copy the block again.

The fix

Write a function that takes the card's ID as an input and handles the setup. Then call it once for each card. Now the behavior lives in one place.

Try it: comment out the WET code and uncomment the DRY version below it. The page works exactly the same — but the code is shorter and easier to change.

Why this matters

Functions aren't just about saving keystrokes. When behavior lives in one place, there's only one place where bugs can hide, and only one place you need to update when something changes.
