JavaScript Targeting elements and adding event listeners

So far, we’ve had our *developer* hats on. How do we write JavaScript in a way that’s useful to the user?

We can run JavaScript on demand when we interact with an HTML element. Here’s how we do that:

  1. Use document.querySelector("") to target an element. If we’re targeting an element by class, we can say something like ".className", or if we’re targeting by ID we can say "#className". The difference is whether we use a period or a hashtag/number sign.
  2. Assign that element to a variable.
  3. Use .addEventListener() to run code when the element is interacted with.

There are a few special things about the function we’re using here:

  • It doesn’t return anything. Instead, it just runs some code. Anything a function does that it’s returned anything is called a side effect. In this case, we’re using the alert() built-in function to cause an alert dialog to appear.
  • This function doesn’t have a name. That means we can’t refer to it from anywhere else. A function without a name is called an anonymous function.
