Loops “while” Loops

Loops allow us to run the same snippet of code multiple times instanteously. This is different from setInterval() because loops run without a delay and don’t run infinitely many times.

There are a few ways to create loops depending on the context and your preference. while loops run until a specified condition is no longer true. As long as the condition is true, the loop will run forever.

We often use loops with a counter variable of some sort. This variable tracks how many times the loop has run. We increase (a.k.a. increment) the counter by 1 each time the loop runs and end the loop when it reaches a specific number.

WARNING: Avoid situations with infinite loops at all costs! These happen when the condition never becomes false. If that happens to you in DEMOLAND, you’ll have to close your browser tab and reopen the demo to fix it. To avoid an infinite loop, you’ll often want the following:

  1. A loop counter variable.
  2. A condition that checks the current value of your loop counter.
  3. A line of code that increments your loop counter each time the loop runs.
