Most everything in a regular HTML document with CSS styling flows "normally." Boxes that have display set as "inline-block" flow one after another. Boxes displayed as "block" will stack one after another. Boxes within a parent container set with the display of "flex" will follow the specified flex box rules. Boxes within a parent container set with the display of "grid" will follow the specified row and column rules. Even specifying relative positioning will preserve the space that the element was originally supposed to be in. But, there are two special positioning properties like absolute and fixed that will take an element out of the normal flow. When that happens, it is as if that element was not there at all. Keep this especially in mind when you are using absolute and fixed child elements within a parent that has display set as flex or grid.
